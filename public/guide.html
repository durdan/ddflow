üìä Flowchart

type: flowchart

Create process flows with different node types and connections.

Syntax

(start) Start node (rounded)
(process) Process/action node
(decision) Decision diamond
(io) Input/Output node
(end) End node (rounded)
A -> B Connection between nodes
A -> B: label Connection with label

Example

(start) Start
Start -> (process) Validate Input
Validate Input -> (decision) Is Valid?
Is Valid? -> (process) Process Data
Is Valid? -> (io) Show Error
Process Data -> (end) Done
Show Error -> (end) Done

üîÑ Sequence Diagram

type: sequence

Visualize interactions between participants over time.

Syntax

participant A, B, C Define participants
A -> B: message Solid arrow (request)
A --> B: message Dashed arrow (response)

Example

participant Client, API, Database, Cache

Client -> API: Login Request
API -> Cache: Check Session
Cache --> API: Not Found
API -> Database: Validate Credentials
Database --> API: User Data
API -> Cache: Store Session
API --> Client: Auth Token

üóÑÔ∏è ERD (Entity Relationship Diagram)

type: erd

Database schema visualization using SQL CREATE TABLE syntax.

Syntax

CREATE TABLE name (...) Define a table
PRIMARY KEY Mark primary key field
REFERENCES table(col) Foreign key relationship

Example

CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255),
  name VARCHAR(100),
  created_at TIMESTAMP
);

CREATE TABLE orders (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  total DECIMAL(10,2),
  status VARCHAR(50),
  created_at TIMESTAMP
);

CREATE TABLE order_items (
  id UUID PRIMARY KEY,
  order_id UUID REFERENCES orders(id),
  product_name VARCHAR(200),
  quantity INT,
  price DECIMAL(10,2)
);

üîÄ State Machine

type: state

Syntax

(final) State Final state
State Regular state
A -> B: EVENT Transition with event

Example

Processing

üì¶ Class Diagram

type: class

Object-oriented class structures with properties and methods.

Syntax

class ClassName Class declaration
+ property: type Public property
- property: type Private property
# property: type Protected property
+ method(params): return Public method

Example

class User
  + id: string
  + email: string
  + name: string
  - passwordHash: string
  + login(email, password): boolean
  + logout(): void
  # validatePassword(plain): boolean

class Order
  + id: string
  + userId: string
  + items: OrderItem[]
  + status: OrderStatus
  + total: number
  + addItem(item): void
  + removeItem(itemId): void
  + calculateTotal(): number

class OrderItem
  + id: string
  + productId: string
  + quantity: number
  + price: number

üèõÔ∏è C4 Diagram

type: c4

C4 model for software architecture visualization.

Syntax

[person] Name: Desc Person/user
[system] Name: Desc Software system
[container] Name: Desc Container

[external] Name: Desc External system
A -> B: Action Relationship

[person] Admin: System administrator

[external] SendGrid: Email delivery

React Frontend -> Node.js API: API calls
Node.js API -> PostgreSQL: Reads/Writes
Node.js API -> Stripe: Processes payments
Node.js API -> SendGrid: Sends emails
Node.js API -> AWS S3: Stores files

üìã Requirement Diagram

type: requirement

System requirements documentation.

Syntax

requirement Name {
  text: Description
  risk: Low | Medium | High
  priority: Low | Medium | High | Critical
}

Example

requirement User Authentication {
  text: Users must authenticate using email and password
  risk: High
  priority: Critical
}

requirement Password Security {
  text: Passwords must be hashed using bcrypt with salt
  risk: High
  priority: Critical
}

requirement Session Management {
  text: Sessions expire after 24 hours of inactivity
  risk: Medium
  priority: High
}

requirement Data Encryption {
  text: All sensitive data must be encrypted at rest
  risk: High
  priority: High
}

requirement Audit Logging {
  text: All user actions must be logged for audit
  risk: Low
  priority: Medium
}
